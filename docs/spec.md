# Aorazer 仕様書

更新日時: 2026-03-31

## 1. 基本コンセプト

* 名前は **Aorazer**
* 読みは **アオレイザー**
* モチーフは **天を仰ぐ / 双眼 / 上向きのまなざし**
* 左右対称性と、身体性を損なわない入力体験を重視する

---

## 2. 初号機の方針

* **有線 split** 前提
* **無線は後回し**
* **中央拡張モジュール構想はいったん捨てる**
* 中央は

  * **磁石による左右連結**
  * 必要なら **受動アタッチメント（Alice角など）**
    までに留める
* 中央の能動拡張は将来課題
* トップ/ボトムプレートやケースは初号機では必須ではなく、まずは動作優先で進める

---

## 3. 構成

* 左右分割キーボード
* **片側1MCUずつ**
* 左右は **TRRS** で接続
* 左右連結用に **磁石** を使う
* split 通信は **serial**
* **USB接続された側が master**
* 初号機は **身体性と配列成立を優先** し、複雑な機構は後回しにする

---

## 4. MCU / ファーム方針

* ファームは **QMK**
* 設定ツールは **Vial**
* **Mod Tap対応**
* MCUは **RP2040系本命**
* 無線化するなら将来別リビジョンで検討
* `nRF52840 + ZMK` は将来の無線版候補だが、初号機では採らない

---

* 通常キー数: **39**

### 行列

## 6. 操作子

### ジョイスティック

* 左右それぞれに **1個ずつ**
* **Nintendo Switch互換品** を採用候補
* **押し込みあり**
* 方向入力と押し込みは QMK / Vial 管理前提

#### 通常時

* 左: **マウス移動**
* 左押し込み: **左クリック**
* 右: **上下左右スクロール**
* 右押し込み: **右クリック**

#### キー入力モード

* **Fn押下中**
* **専用レイヤー時**

##### 左ジョイスティック

* 上: 上選択
* 下: 下選択
* 左: 左単語選択
* 右: 右単語選択
* 押し込み: 選択解除

##### 右ジョイスティック

* 上: 上移動
* 下: 下移動
* 左: 左単語移動
* 右: 右単語移動
* 押し込み: なし

> 注: 上記ジョイスティック仕様は確定方針であり、初号機ファームでは後回しにする可能性がある。

### エンコーダー

* 左右それぞれに **1個ずつ**
* **人差し指で回す位置** に再配置
* 押し込みも使用する

#### 左エンコーダー

* 時計回り: アプリ切り替え →
* 反時計回り: アプリ切り替え ←
* 押し込み: アプリ切り替え確定

#### 右エンコーダー

* 時計回り: タブ切り替え →
* 反時計回り: タブ切り替え ←
* 押し込み: タブを閉じる

## 7. LED

* RGBバックライトあり
* LEDは **SK6812 MINI-E**
* HaLIK用の余剰分を使う前提
* 目的は

  * **レイヤー状態表示**
  * **有効なキーのみ点灯**
* 初号機では、少なくとも **レイヤーごとの色変更** を目標にする
* 左右は物理的には別MCU駆動だが、**論理挙動として同一制御** を目指す

---

## 8. レイアウト

* 最新の JSON レイアウトを現行版とする
* `Fn` キーと `Delete` キーは入れ替え済み
* `◇` キーは

  * **Vialで自由に触る前提の空きキー**
  * **レイヤー切り替え用途候補**

---

## 9. Emacsキー仕様

### 配置

* 左中段の `Ctrl` 位置を **Emacs** キーに変更

### 役割

* **OS差を吸収する主修飾キー**
* 編集系ショートカットは Emacs 風に振る舞う
* それ以外は

  * **Windowsでは Ctrl**
  * **Macでは Command**
    として振る舞う

### 編集系

* `Emacs + A` → 行頭
* `Emacs + E` → 行末
* `Emacs + H` → Backspace
* `Emacs + B` → Left
* `Emacs + F` → Right
* `Emacs + P` → Up
* `Emacs + N` → Down

### 変換系

変換系は IME 固有キーではなく、**F6〜F10送出** に統一する。

* `Emacs + J` → `F6`
* `Emacs + K` → `F7`
* `Emacs + L` → `F8`
* `Emacs + ;` → `F9`
* `Emacs + '` → `F10`

対応イメージ:

* J = ひらがな
* K = 全角カタカナ
* L = 半角カタカナ
* ; = 全角英数
* ' = 半角英数

### 補足

* `Ctrl+F` など既存ショートカット競合は、**左下の通常Ctrlキー**で解決する前提
* 以前の `Emacs + K = カーソル以降削除` は廃止し、変換列の連続性を優先する

---

## 10. OS判定

* **QMKの OS Detection を使う前提**
* `Emacs` キーの

  * Win時 Ctrl
  * Mac時 Command
    の切替に使う
* 手動 OS 切替キーも Fn レイヤー側に持てる構造を想定する
* OS状態保存は **再起動ごとの自動判定** を基本とする

---

## 11. Fnレイヤー

* `Fn + 1-0` → `F1-F10`
* `Fn + -` → `F11`
* `Fn + =` → `F12`
* `Fn + ↑ / ↓` → バックライト明るさ変更
* `Fn + ← / →` → RGBカラー切り替え

### 追加候補

* OS切替
* RGB ON/OFF
* メディア再生停止
* 音量

---

## 12. Vialまわり

* 設定ツールは **Vial**
* まずは **QMK default keymap の最小成立** を優先し、その後に Vial 対応を追加する
* Vial 側では

  * レイヤー変更
  * RGB設定
  * 空きキーの再配置
    を触りやすくする方針
* Emacsキー挙動は `keymap.c` 側で固定する

---

## 13. 初号機の実装優先順位

* **初号機はできるだけ軽くする**
* **身体性と配列の成立を優先**
* **無線・中央拡張・複雑な機構は先送り**
* **QMK + RP2040 + TRRS + 磁石連結** が中核
* 設定系は **Vial** に寄せる
